Audience: internal planning + alignment with Magy. Status: drafted from Magy's Slack replies on 2026-05-02. Target ship: before Tuesday 2026-05-05 11:40 demo with HCI 271.

## 1. Positioning: a third PromptDesigner mode

LEAI PromptDesigner currently has two modes (see `PromptDesigner.html` lines 318–328 and 581–583):

1. **General Course Feedback** (`data-mode="general"`) — open-ended feedback bot.
2. **In-Group Feedback** (`data-mode="group"`) — team-aware bot with team configuration, used for cross-teammate feedback.

**Add a third mode: `data-mode="form"` — "Form-Mapping Reflection".** Naming candidates:

- *Form-Mapping Reflection* (clear about the contract)
- *Structured Reflection* (clear about UX)
- *Reflection Form Mode*

Default to **"Structured Reflection"** in the UI; reference name `form` internally. Keeping it as a separate mode is the key compatibility move — the existing two modes (and every other class using LEAI) stay untouched.

## 2. What "form mode" guarantees that the other modes don't

| Property                                          | General | In-Group | Form (new) |
| ------------------------------------------------- | ------- | -------- | ---------- |
| Conversational bot                                | yes     | yes      | yes        |
| Team-aware (team picker, per-team sessions)       | no      | yes      | **yes** (reuses In-Group infra) |
| Bound to a specific written form/template         | no      | no       | **yes**   |
| Coverage enforcement (every section must be hit)  | no      | no       | **yes**   |
| Per-section "anything else?" transition           | no      | no       | **yes**   |
| One-pass deeper probe on shallow answers          | soft    | soft     | **enforced once** |
| Structured form-mapped download                   | no      | no       | **yes**   |
| Raw transcript download                           | partial | partial  | **yes** (always) |
| TTS audio summary per submission                  | no      | no       | **yes** (post-MVP within Tuesday window if time) |
| Insights report across submissions                | no      | partial  | **yes**   |

## 4. Final Tuesday scope (covers everything Magy asked for)

Numbered for tracking; (5) and (6) are the audio pieces, sequenced last.

## 5. Workflow Magy + students experience

**Instructor (Magy / TA), one-time setup:**

1. Open PromptDesigner → choose **Structured Reflection** mode.
2. Select course + week (HCI 271, Week 6).
3. Configure teams (team count + sizes; no names needed at config time, students pick when they enter).
4. Click "Use Week 6 HCI 271 Reflection Template" — the Week 6 reflection prompt and section schema load. (For future weeks/courses, the template registry is extensible.)
5. Save → get one survey link per team.
6. Paste the auto-generated **Canvas instructions paragraph** (see §7) into the Canvas assignment.

**Student, in class on Tuesday:**

1. Click their team's link from Canvas.
2. Pick their team membership from the dropdown.
3. Chat with Remi — the bot walks them section-by-section through Parts 1, 2, 3.
4. End conversation → "Download your reflection" button appears → PDF (structured form on top, raw transcript below) downloads.
5. Upload the PDF as the Canvas assignment submission.

**Instructor reviews:**

1. Open FeedbackAnalyzer → select the Week 6 survey.
2. Per submission: read the structured insights report, optionally play the TTS audio.
3. Cross-submission: see topic-frequency rollup (which themes show up most, which sections were thinly answered class-wide).

## 6. Architecture sketch (compatible with existing setup)

- **Frontend:**
  - `PromptDesigner.html` — add `data-mode="form"` tab next to the existing two; new panel `#mode-panel-form` with template-picker, team config (reused from In-Group), and a read-only preview of the section schema.
  - `FeedbackChat.html` (and the underlying chat wrapper in `leai-shared.js`) — gain a "form mode" path that consults a section schema, tracks coverage state per section, and gates the closing `[END]` token until every section has at least one student response.
  - `FeedbackAnalyzer.html` — per-submission view shows: structured form output, raw transcript, insights report, audio playback. Survey-level view shows topic-frequency rollup.
  - **New:** `LEAI/docs/forms/hci271-week6-reflection.json` — the section schema (10 prompts: 1.1, 1.2, 1.3a/b/c, 1.4, 2.1, 2.2, 2.3, 2.4, 3.1, 3.2; each with id, title, prompt, depth-probe text, expected fields like the 2.2 table or 2.4 rating tuple).
  - `leai-shared.js` — extend `leaiChat` with `enterFormMode(schema)` and a `coverageTracker` helper; extend `leaiAnalysis` with a "structured extraction" pass that maps the transcript back onto the schema for the download.
- **Backend (`guiidatapipelines`):** existing `openai-chat/` and `message/` endpoints already log everything we need. New endpoints (or extend existing):
  - `form_schemas/` — list available form schemas (Week 6 to start).
  - `submission_export/?session_id=…&format=pdf|md` — server-rendered structured + transcript export. *Alternative for Tuesday: render PDF client-side to avoid a backend deploy on the critical path.*
  - `submission_insights/?session_id=…` — return cached or freshly-computed insights report; TTS endpoint wraps OpenAI TTS.
- **Data model:** add `survey.mode` (`general` | `group` | `form`), `survey.form_schema_id`. Existing team-config tables already cover the team side. Submission record gains `structured_payload_json` (for the form-mapped output) and `insights_report_md`.
